## User Requirement - Backend As A Service

### Phase-01

**Authentication & Roles**

UR-01 : Users shall be able to log in to the dashboard using an email and a password

UR-02 : First user to log in to the dashboard shall be super-user of the dashboard

UR-03 : Only super-user shall be able to create new users, delete users, modify user information for users in dashboard.

UR-04 : Super-user shall be able to create rules other users to follow.

UR-05 : Super-user should be able to create roles to restrict access, action that can be taken within dashboard.

UR-06 : Users should be able to change there email and password.

---

**Data Management**

UR-07 : Users should be able to create, delete ,update and access data via dashboard.

UR-08 : Super-user or users with access to data source should be able to set rules, that every user shall follow when accessing data.

UR-12 : Users should be able to see documentation about how to interact with data source.

UR-13 : Users should be able to explore data in a data table via dashboard.

UR-14 : Users should be able to see relationships between data sources via dashboard.

---

**External Access**

UR-09 : External application shall be able to access data sources if they have access.

UR-10 : External applications or their users shall not able to access dashboard.

UR-11 : External application shall only access data sources, exposed by the system to outside.
